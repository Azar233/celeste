# Celeste Rust 课程项目

这是一个基于 Rust 与 Bevy 0.15.1 的 2D 平台跳跃游戏练习项目，目标是逐步复现类似 Celeste 的角色手感、移动机制和像素风格表现。

当前项目还处在原型阶段，但已经完成了角色移动、跳跃、墙壁交互、冲刺、基础动画切换、头发表现和冲刺光痕等核心模块。整个实现目前主要集中在 src/main.rs 中，方便快速迭代和验证手感。

## 项目目标

这个项目的核心目标不是先堆完整关卡内容，而是优先把 Celeste 风格里最关键的几个部分做出来：

- 角色横向移动的响应速度
- 起跳、短跳、下落的手感差异
- 贴墙、爬墙、蹬墙跳的交互逻辑
- 冲刺的方向、时长和视觉反馈
- 角色头发、刘海和冲刺状态之间的风格统一

换句话说，这个项目优先解决的是“角色操作和视觉反馈是否成立”，而不是“地图内容是否丰富”。

## 当前已实现内容

- 基础 2D 相机与固定纵向视口
- 玩家水平移动、地面摩擦、空中控制
- 可变跳跃高度
- 下落加速与短跳截断
- 墙滑、抓墙、爬墙、墙跳
- 八方向输入冲刺
- 冲刺剩余次数控制与重置
- Idle / Run 贴图动画切换
- bangs.png 刘海贴图与角色主体分层渲染
- 程序化后发梢模拟
- 头发黑色描边与像素化 shader
- 冲刺白色像素光痕
- 简单地面、墙体和死亡重生逻辑

## 资源结构

目前使用到的主要资源如下：

- assets/idle_sheet.png：角色待机动画图集
- assets/run_sheet.png：角色奔跑动画图集
- assets/bangs.png：刘海贴图，使用白底方便通过颜色着色切换发色
- assets/shaders/hair_outline.wgsl：后发梢像素化黑描边 shader

## 运行方式

在项目根目录执行：

```bash
cargo run
```

注意：需要从项目根目录运行。直接运行 target/debug 下的可执行文件会导致 assets 路径解析错误。

## 操作方式

- A / D：左右移动
- W / S：上 / 下方向输入
- Space：跳跃
- J：抓墙 / 爬墙
- K：冲刺

## 当前制作过程与实现思路

### 1. 先做角色手感，而不是先做完整关卡

最开始的重点是先把角色控制做出来，因此项目最早实现的是：

- 速度与重力参数
- 地面和空中的加速度差异
- 碰撞检测与位置修正
- 跳跃和冲刺这些最基础的状态机

这一步的思路是，Celeste 这类游戏最重要的不是“贴图像不像”，而是“按键后角色是否立即给出正确反馈”。如果移动手感不成立，后面的动画和特效价值也会很低。

### 2. 用最小原型搭出墙壁交互

在基本移动完成后，继续加入了：

- 墙壁接触方向检测
- 墙滑速度上限
- 抓墙后上下爬墙
- 中立墙跳与带方向输入的墙跳

这部分本质上是在给玩家提供“空中纠错”和“连续位移”的能力，也是 Celeste 操作体验里最核心的部分之一。

### 3. 用图集切换补齐角色表现

在操作逻辑稳定后，加入了 idle_sheet 和 run_sheet 两套图集。角色根据当前状态在 Idle / Run 之间切换，同时按朝向决定是否 flip_x。

这里没有引入复杂动画系统，而是直接通过 TextureAtlas 的 index 递增来控制帧切换，优点是：

- 结构简单
- 易于和当前原型整合
- 足够支撑像素角色动画验证

### 4. 头发分成“前发贴图 + 后发程序化”两层

头发部分没有完全走单一方案，而是拆成了两部分：

- bangs.png：作为前发 / 刘海，直接用 Sprite 渲染
- 程序化后发梢：用 5 段简单链状点位模拟，并通过自定义材质渲染成像素块

这样拆分的原因很实际：

- 刘海对角色正面轮廓影响很大，用贴图更容易控制造型
- 发梢更适合跟随速度和冲刺状态摆动，用程序化方法更灵活
- 两者结合后，既能保留像素角色正面辨识度，也能获得动态反馈

### 5. 冲刺反馈不只靠速度，还要靠光痕

冲刺本身已经通过方向向量和固定时长建立了手感，但仅靠速度变化不够明显，因此又增加了白色像素光痕。

这部分没有引入完整粒子系统，而是用轻量方式实现：

- 冲刺时按固定时间间隔发射一批短寿命像素粒子
- 这些粒子沿冲刺反方向排成细线
- 粒子会快速淡出并略微位移

这样做的优点是实现简单、可控，而且很贴合当前像素风原型。

## 关键算法与技术实现

### 1. AABB 轴向碰撞检测

项目当前使用的是最直接的 AABB 碰撞检测：

- 角色和地面 / 墙体都视为矩形
- 分别处理 X 轴与 Y 轴位移
- 发生碰撞后只在当前轴上回退并修正速度

这种“分轴移动 + 分轴修正”的方法非常适合平台跳跃原型，因为它：

- 简单稳定
- 便于判断角色是在落地、撞墙还是顶头
- 很容易扩展到墙滑、墙跳等状态逻辑

### 2. move_towards 加速度控制

水平移动没有直接把速度设置成目标值，而是通过 move_towards 逐渐逼近目标速度。这样可以自然表现：

- 起步加速
- 松手减速
- 反向时更强的转向摩擦

这也是当前角色“有重量感但不拖沓”的主要来源。

### 3. 可变跳跃与下落倍率

跳跃不是单一重力曲线，而是区分了几种情况：

- 上升阶段松开跳跃键：使用更大的 LOW_JUMP_MULTIPLIER，形成短跳
- 下落阶段：使用更大的 FALL_MULTIPLIER，加快下落

这种做法是平台跳跃手感里非常常见的技巧，可以显著提升控制精度。

### 4. 墙壁状态与墙跳逻辑

墙壁系统依赖两个部分：

- 角色左右侧的碰撞感应
- 一个短暂的 WallJumpTimer 锁定输入

这样可以避免墙跳刚触发就立刻被横向输入抵消，同时能区分：

- 抓墙爬墙跳
- 中立墙跳
- 有方向输入的蹬墙跳

### 5. 冲刺状态机

冲刺通过 DashState 管理：

- is_dashing：当前是否处于冲刺中
- timer：剩余冲刺时长
- direction：冲刺方向向量
- dashes_remaining：剩余冲刺次数

当冲刺触发后，角色速度直接设置为方向向量乘 DASH_SPEED；计时结束后关闭冲刺，并把速度乘上一个结束倍率，避免停得过于生硬。

### 6. 程序化头发链条模拟

后发梢不是刚体，而是一个简化的“链条”：

- 第一个点固定在头发根部
- 后续点逐个跟随前一个点
- 每个点都会被拉向一个静态参考位置 rest_target
- 同时受到简单风力 / 重力偏移
- 最后通过距离约束把点与点之间保持在固定长度附近

这其实是一种很轻量的约束链思路，优点是：

- 不需要完整物理引擎
- 可控性强
- 很适合做像素角色的发梢、披风、尾巴之类的二次动态

### 7. 头发像素化描边 shader

后发梢使用自定义 2D 材质和 WGSL shader 渲染，而不是直接贴普通圆形贴图。shader 的主要思路是：

- 先把 UV 量化到较粗网格上，制造像素化边缘
- 再计算圆形距离和方形距离
- 用两者混合出一个更适合像素风的形状边界
- 用内外两层阈值差值得到黑色描边区域

最终得到的效果是：

- 发梢整体是像素块质感
- 外缘有清晰黑描边
- 发色可通过 uniform 动态切换

### 8. 冲刺光痕生成算法

当前冲刺光痕的实现逻辑是：

- 获取当前冲刺方向向量
- 取其反方向作为光痕延伸方向
- 以固定间隔在角色后方生成多个短寿命小像素点
- 将这些点排成一条线
- 每帧衰减透明度，并在生命周期结束时自动销毁

这种方式本质上是“手写的简化粒子系统”，优点是足够小而且足够像素风。

## 当前代码结构特点

目前项目还没有做模块拆分，绝大多数逻辑都在 src/main.rs 中。这样做的主要原因是当前阶段还在频繁调手感和视觉参数，集中在一个文件里修改会更快。

后续如果项目继续扩大，比较自然的拆分方向会是：

- player.rs：玩家状态、输入和运动
- collision.rs：碰撞与地图交互
- hair.rs：头发模拟与 bangs 逻辑
- effects.rs：冲刺光痕和后续特效
- animation.rs：角色动画切换

## 目前的局限

到目前为止，项目仍然属于原型验证阶段，还存在一些明显未完成部分：

- 没有正式关卡数据结构
- 没有真正的瓦片地图系统
- 没有音效和镜头反馈
- 没有敌人、机关和可收集物
- 玩家状态仍然比较集中地耦合在单文件里

这些都不是当前阶段的阻塞问题，因为现阶段目标仍然是先把角色操作和视觉反馈打磨到更接近 Celeste 风格。

## 下一步可以继续做什么

如果继续沿着当前方向推进，比较合理的下一阶段工作会是：

- 引入真正的关卡结构或 tilemap
- 给冲刺、跳跃、落地增加更强的镜头 / 屏幕反馈
- 加入死亡、重生点与房间切换
- 把角色状态逻辑从 main.rs 中拆分出去
- 调整头发与冲刺特效，使视觉更接近原作节奏
